🏃 数字运动康复师 - AI运动损伤智能问诊系统

✨ 核心亮点

🎯 精准问诊流程

智能问诊门卫：在检索前主动交互，确保获取疼痛位置、痛感描述、发生场景三大关键信息，从源头杜绝“幻觉回答”

场景化知识路由：自动识别损伤部位（膝/肩/腰等），智能筛选对应知识库，提升回答相关性

五段式专业回答：严格遵循损伤定性→即刻处理→康复练习→风险警示→免责声明的医疗安全框架

🏗️ 完整数据流水线

原始PDF → 专业清洗 → AI自动标注 → 向量化入库 → 智能检索

自动化元数据标注：利用LLM为文档智能打标（category、doc_type、priority）

医学文档特化清洗：修复中文排版问题，过滤页码/页眉等噪音

元数据驱动检索：基于文档分类实现精准知识筛选

🏥 系统架构
核心模块

Frontend (Streamlit)

        ↓
        
[问诊门卫 Agent] → 信息不全 → 主动追问

        ↓ (信息完整)
        
[意图识别模块] → 确定损伤类别 → 筛选知识库

        ↓
        
[RAG检索引擎] → 多源文档检索 → 结果融合

        ↓

        ↓
        
[前端展示] ← 答案+依据+质量评分

技术栈

模块--------技术选型----------说明

前端交互​--Streamlit---快速构建医疗级Web界面--支持对话历史、文件上传

RAG框架​--LangChain--模块化构建检索链，支持自定义工作流

向量引擎​--Chroma--轻量级向量数据库，支持元数据过滤

嵌入模型​--DashScope Embeddings--中文优化的文本嵌入

大语言模型​--通义千问 API--用于生成、问诊、标注等多任务

数据处理​--PyPDFLoader + 自定义清洗--处理医学PDF/文本的完整管道

🚀 快速开始

1. 环境配置

git clone https://github.com/yourusername/sports-rehab-ai.git
cd sports-rehab-ai
pip install -r requirements.txt

2. 配置API密钥

# 在 .env 文件中配置
DASHSCOPE_API_KEY=your_dashscope_key_here

3. 构建知识库

# 将运动医学指南PDF放入 ./sports_data/
python data_cleaner.py        # 清洗文档
python auto_labeler.py        # AI自动标注
python build_knowledge_base.py # 构建向量库

4. 启动系统

streamlit run app.py

🙋 常见问题

Q：需要医学背景才能使用吗？​

A：不需要。系统会引导你提供关键信息，并给出通俗易懂的建议。

Q：能替代医生吗？​

A：绝对不能。本系统是健康科普与初步指导工具，所有严重损伤必须就医。

Q：支持哪些文档格式？​

A：目前支持PDF和TXT格式的运动医学指南、康复论文等。
